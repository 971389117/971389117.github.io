原型与原型链,prototype和__proto__

规范的获得与设置对象原型

实际应用中，应避免使用 __proto__

原型与原型链

每个对象都有一个proto属性，这个不是规范的，而是每个浏览器自己造的，它和[[prototype]]其实是一个意思。

查找一个属性时，对象先在自己的属性中查找，如果没找到，再到[[prototype]]中查找，如果还没有再向上一层[[prototype]]中查找，直到 Object.prototype 若还没有就返回 undefined，这就是 JS 的原型与原型链

prototype 和proto

  1. 每个对象都有一个 proto 属性，该属性指向该对象构造函数的 prototype 指向的对象，也就是原型对象

呢么，函数的的构造函数是谁呢？

即便是看起来的矛盾体，也不例外

对象原型的终点都是 Object.prototype

练习 1.

proto 和 prototype 的区别是什么？ 1.不能断章取义，proto 和 prototype 只是两个 key 而已 2.我们一般研究对象的proto和函数的 prototype 3.对象.proto === 某函数.prototype 4.如果把函数看成对象，那么函数.proto === Function.prototype 5.如果把 Function 看成对象，那么 Function.proto === Function.prototype

练习 2.

var obj = {

name: 'frank'

}
请问 obj.toString 为什么不是 undefined？ 1.因为 obj 以某种方式拥有了 toString 这个属性
2.obj 拥有 toString 的方式是用过 proto 达到的
3.obj.proto 存储了 Object.prototype 的地址（也就是引用关系）
4.obj.toString === Object.prototype.toString 为 true

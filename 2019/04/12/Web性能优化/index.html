Web性能优化

web 性能优化

什么是 web 性能优化呢？用户感觉 网站加载贼快！
CPU 占用少属于 服务器优化

资源的消耗总是存在的，只是对妥协现实与技术进行妥协，权衡比重，达到一个体验上的最佳感觉

网页加载步骤与优化

  1. 缓存
  2. DNS 查询
    【减少 DNS 查询】
  3. 建立 TCP 连接 X
    【连接复用 keep-alive | HTTP2.0 多路复用 | HTTP3.0】
  4. 发送 HTTP 请求 ETag
    【减小 cookie | cache - control |
    增加域名数量：同时发送多个 HTTP 请求 CDN
    并发：一个域名 4 或 8 个 根据浏览器不同
    】
    》后台处理（发送响应）
  5. 接受响应
    【 ETag 304 1.css/xxx | gzip 压缩 】
  6. 接收完成（readyState） -> HTML
  7. DOCTYPE html/xml 【不能不写】
  8. 逐行解析（阅读） HTML，
  9. 看到<h1>
    IE CSS 不阻塞渲染 1.在页面显示标题
    Chrome 阻塞渲染 2.不在页面显示标题
    【减少标签 - 】
  10. 看到 css
    下载 css
    同时下载 css2
    同时下载 css3
    ……
    注：下载并行，解析串行
    【CDN】
  11. 看到 js
    下载 js
    同时下载 js2
    同时下载 js3
    ……
    注：js 阻塞 HTML 渲染
    【CDN】

other：延迟加载（懒加载） | 预加载 | 合并文件 | loading | optimize image

问：关于第二步的减少 DNS 查询和第四步的增加域名数量冲突了，怎么办？
答：根据 HTTP 请求的数量决定，例如，只有两个 HTTP 请求，那么没必要增加域名数量，但是如果有十多个二十多个 HTTP 请求，增加域名数量提高并发就有必要了。

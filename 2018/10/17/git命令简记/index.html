前情提要篇

万物第一件事是安装与更新，但是网络教程繁多，本文目的为自己知识整理掌握，不一一详述。

问：如何设置 SSH_KEY?
答：
官网解答：设置ssh key

自己解答：

  1. 在本机生成 sshkey

运行 ssh-keygen -t rsa -b 4096 -C "邮箱"
运行 cat ~/.ssh/id_rsa.pub 复制显示的 sshkey

  1. 在 GitHub 上设置 sshkey

注册账户,右上角选择 settings, 选择 SSH and GPG keys

  1. 回到命令行
    运行 ssh -T git@github.com
    如果告诉你”Hi 971389117! You’ve successfully authenticated, but GitHub does not provide shell access.”,就成功了,否则重新来过.

问：如何配置本地 GitHub 账户信息？
git config –global user.name 你的英文名
git config –global user.email 你的邮箱
git config –global push.default matching
git config –global core.quotepath false
git config –global core.editor “vim”
五句话，依次运行。不执行的话，电脑可能会爆炸你信不信。

查看当前生效配置 git config -l
其它可以参考 tldr git config

基础入门

应用题:(一)在桌面生成一个文件夹,进入里面,初始化 git 仓库,生成两个任意文件,添加到暂存区,最后提交修改.每个步骤都要查看状态.

解答:

初始化仓库 git init
添加到暂存 git add *
提交 git commit -m "评论"
查看状态 git status -sb s 不是summary 摘要,总结,而是 short; b是 branch 分支
查看日志 git log

(二)在GitHub 上新建一个仓库,名称随意,将本地项目与远端项目链接起来.
git remote add origin git@github.com:971389117/git-demo-1.git

(三)将本地仓库推送到远程仓库
答:1.git add *

2. git commit -m ""
3. git pull 将远程拉下来（更新本地仓库）
4. git push 或者 git push -u origin master
